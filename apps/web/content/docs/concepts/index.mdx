Understand the core building blocks that make up OwlMetry.

- **[Events](/docs/concepts/events)** -- The core data unit. Log levels, sessions, timestamps, deduplication, and custom attributes.
- **[Structured Metrics](/docs/concepts/metrics)** -- Lifecycle operations (start/complete/fail/cancel), single-shot measurements, duration tracking, and aggregation queries.
- **[Funnels](/docs/concepts/funnels)** -- Multi-step conversion tracking with open and closed analysis modes, drop-off rates, and experiment segmentation.
- **[A/B Experiments](/docs/concepts/experiments)** -- Client-side variant assignment with persistent storage and automatic event tagging.
- **[Projects & Apps](/docs/concepts/projects-and-apps)** -- How projects group apps across platforms, bundle ID validation, and auto-created client keys.
- **[Authentication & API Keys](/docs/concepts/authentication)** -- Passwordless email auth, client vs agent keys, permissions, team roles, and invitations.
- **[Data Mode](/docs/concepts/data-mode)** -- Filtering between production and development data across the dashboard, CLI, and API.
